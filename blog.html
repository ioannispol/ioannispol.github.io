Virtual environments

Developing Machine Learning (ML) and Deep Learning (DL) projects has become more common than ever before. ML and DL is literary everywhere nowdays, from big corporations to schools and universities, and used in areas from medicine to robotics. But with the extensive use and variability of the projects where ML/DL is used, there are problems need to tackle with. The most common issue arises when working and developing such projects is the conflict between different libraries. This problem can easily be solved if we use virtual environments.

A virtual environment is exactly what the word describes, an environment, a sandbox, where we install libraries and software particular to the project we're working on. In the world of data science, ML and DL, the most well known platform used to create and manage virtual environments is the Anaconda environments.

Anaconda environments offering a simple and elegant solution to mange our environments based on the requirements of the project we're working on. I think the power of anaconda environments comes from the fact that we can use different version of python from a single installation. Also, we can easily use different anaconda repositories to install the GPU versions of ML/DL frameworks such as TensoFlow and PyTorch with ease.

Installation

Here, the miniconda a minimal version of Anaconda will be install. Download the latest miniconda Linux installer script at https://repo.anaconda.com/miniconda/Miniconda3-latest-Linux-x86_64.sh. Also, at Miniconda â€” Conda documentation different miniconda installers and for different platforms (Windows, MacOSX, Linux).

Once the installer downloaded, open a terminal and navigate the the location the script has been download. For me is in /home/myusername/Downloads First, check and verify the integrity of the installer

sha256sum Miniconda3-latest-Linux-x86_64.sh

and will get something similar to 4ee9c3aa53329cd7a63b49877c0babb49b19b7e5af29807b793a76bdb1d362b4 this value need to compare it with the one given on the miniconda site. If the values much means the download is not corrupted and can continue with the installation.

Then need to change make the script executable (if it is not already) using the chmod command and then run the installer itself with the ./program-to-install This can be done by running the following command

chmod +x Miniconda3-latest-Linux-x86_64.sh && ./Miniconda3-latest-Linux-x86_64.sh

The installation procedure is pretty clear after that point. Need to scroll down to the end of EULA terms and conditions and type yes to agree with them.

Also it is a good idea to agree with the default installation path witch is normally under the path /home/myusername/miniconda3. The installation directory can also be specified to a different location but need to add it to the path variable on your system to run miniconda.

The next prompt is important and will ask to initialize miniconda after the installation and need to type yes. If following the default no will need to initialize miniconda manually. To do so need to manually initialize for the shell you are using. For example my shell is zsh so the command will be conda init zsh and then need to restart your shell with

source ~/.zshrc

Now, the last step is not something that need to do, but anaconda activates the base environment whenever you open you terminal by default, and I found that a bit of annoying. So to remove the default functionality need to execute the following

conda config --set auto_activate_base false
